# Browser, DNS, Rendering

## Теория (вопросы и ответы)

### Что происходит, когда пользователь вводит URL в браузере и нажимает Enter?

1. Браузер проверяет DNS-кеш.
2. Если записи нет — обращается к DNS-серверу.
3. DNS возвращает IP-адрес сайта.
4. Браузер устанавливает соединение с сервером.
5. Отправляется HTTP-запрос.
6. Сервер возвращает HTML-документ.
7. Браузер начинает строить страницу.
8. Полученные данные могут быть сохранены в кеш.

---

### Что такое DNS?

DNS (Domain Name System) — система доменных имен.

Она преобразует:

```text
google.com
```

в

```text
142.250.74.14
```

чтобы браузер понимал, на какой сервер отправлять запрос.

---

### Что такое DOM?

DOM (Document Object Model) — объектное представление HTML-документа в виде дерева.

Пример:

```html
<body>
  <h1>Hello</h1>
</body>
```

DOM:

```text
Document
 └─ body
     └─ h1
```

JavaScript работает именно с DOM.

---

### Что такое CSSOM?

CSSOM (CSS Object Model) — объектное представление CSS в виде дерева.

На его основе браузер понимает:

- размеры элементов;
- цвета;
- отступы;
- расположение.

---

### Что такое Render Tree?

Render Tree — объединение DOM и CSSOM.

В него попадают только отображаемые элементы.

Например:

```css
display: none;
```

элемент не попадет в Render Tree.

---

### Какие этапы рендеринга страницы существуют?

#### 1. Построение DOM

Браузер парсит HTML.

#### 2. Построение CSSOM

Браузер парсит CSS.

#### 3. Создание Render Tree

Объединение DOM и CSSOM.

#### 4. Layout (Reflow)

Вычисление:

- размеров;
- координат;
- расположения элементов.

#### 5. Paint (Repaint)

Отрисовка:

- текста;
- цветов;
- теней;
- границ.

#### 6. Composite

Объединение слоев в финальное изображение.

---

### Что такое Reflow?

Reflow (Layout) — перерасчет размеров и положения элементов.

Очень дорогая операция.

Вызывают:

```css
width
height
margin
padding
display
position
font-size
```

---

### Что такое Repaint?

Перерисовка элемента без изменения его размеров и положения.

Например:

```css
background-color
color
border-color
```

Repaint дешевле Reflow.

---

### Почему анимации лучше делать через transform и opacity?

Потому что они:

- не вызывают Reflow;
- обычно не вызывают Paint;
- работают через Composite;
- используют GPU.

Пример:

```css
transform: translateX(100px);
opacity: 0.5;
```

---

### Что такое will-change?

Подсказка браузеру:

```css
.card {
  will-change: transform;
}
```

Браузер заранее создаст отдельный слой и подготовится к анимации.

---

### Что такое семантическая верстка?

Семантическая верстка — использование HTML-тегов по назначению.

Правильно:

```html
<header>
  <nav>
    <main>
      <section>
        <article>
          <footer></footer>
        </article>
      </section>
    </main>
  </nav>
</header>
```

Неправильно:

```html
<div class="header">
  <div class="nav">
    <div class="footer"></div>
  </div>
</div>
```

---

### Зачем нужна семантика?

- SEO
- Accessibility
- Удобство поддержки
- Лучшее понимание страницы браузером

---

### Что такое Accessibility (A11y)?

Accessibility — обеспечение доступности сайта для людей с ограниченными возможностями.

Например:

- поддержка клавиатуры;
- screen readers;
- корректная семантика;
- aria-атрибуты.

---

### Для чего нужен атрибут role?

Позволяет объяснить screen reader назначение элемента.

Пример:

```html
<div role="button">Нажми меня</div>
```

---

### Как работает script без async и defer?

```html
<script src="app.js"></script>
```

Браузер:

1. останавливает парсинг HTML;
2. скачивает скрипт;
3. выполняет скрипт;
4. продолжает парсинг HTML.

---

### Как работает defer?

```html
<script defer src="app.js"></script>
```

Браузер:

1. продолжает парсить HTML;
2. параллельно скачивает скрипт;
3. выполняет после построения DOM.

Сохраняет порядок выполнения.

---

### Как работает async?

```html
<script async src="app.js"></script>
```

Браузер:

1. продолжает парсить HTML;
2. параллельно скачивает скрипт;
3. выполняет сразу после загрузки.

Порядок выполнения не гарантируется.

---

### Разница между async и defer

| async                            | defer                           |
| -------------------------------- | ------------------------------- |
| Выполняется сразу после загрузки | Выполняется после парсинга HTML |
| Не гарантирует порядок           | Сохраняет порядок               |
| Для аналитики                    | Для приложения                  |

---

### Что такое DOMContentLoaded?

Событие возникает после полной загрузки HTML и построения DOM.

Не ждёт:

- картинки;
- видео;
- шрифты.

Пример:

```js
document.addEventListener("DOMContentLoaded", () => {
  console.log("DOM готов");
});
```

## Практические вопросы собеседований

### Что быстрее?

```css
opacity: 0;
```

или

```css
display: none;
```

Ответ:

Зависит от задачи.

Для анимации — opacity.

Для полного удаления из потока — display:none.

---

### Почему анимация через left хуже transform?

Плохо:

```css
left: 100px;
```

Вызывает Reflow.

Хорошо:

```css
transform: translateX(100px);
```

Работает через Composite.

---

### Что произойдет при подключении скрипта в head?

```html
<head>
  <script src="app.js"></script>
</head>
```

Парсинг HTML остановится до выполнения скрипта.

---

### Когда использовать async?

Для независимых скриптов:

- аналитика;
- метрики;
- реклама.

---

### Когда использовать defer?

Для основного JavaScript приложения.

Например:

```html
<script defer src="bundle.js"></script>
```

---

### Что тяжелее: Reflow или Repaint?

Ответ:

Reflow.

Потому что после Reflow часто происходит Repaint.

---

### Какие свойства вызывают Reflow?

Примеры:

```css
width
height
padding
margin
display
font-size
position
```

---

### Какие свойства вызывают только Repaint?

Примеры:

```css
background
color
border-color
visibility
```

---

### Что выведет код?

```js
document.addEventListener("DOMContentLoaded", () => {
  console.log("DOM ready");
});
```

Ответ:

Сообщение появится после построения DOM.

---

### Почему семантическая верстка важна?

Ответ:

- улучшает SEO;
- помогает screen reader;
- делает код понятнее;
- улучшает поддержку проекта.

---

### Назови этапы рендеринга страницы

Правильный порядок:

```text
HTML
↓
DOM
↓
CSSOM
↓
Render Tree
↓
Layout
↓
Paint
↓
Composite
```
