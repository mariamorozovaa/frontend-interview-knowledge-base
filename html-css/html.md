## 6. Когда использовать async и defer?

```html
<!-- ========== Без атрибутов (обычный) ========== -->
<!-- Блокирует парсинг HTML, соблюдает порядок, выполняется сразу после загрузки -->
<script src="script.js"></script>
<!-- HTML парсится → скрипт загружается → скрипт выполняется → HTML продолжает парситься -->

<!-- ========== async ========== -->
<!-- НЕ блокирует парсинг HTML, НЕ соблюдает порядок, выполняется сразу после загрузки -->
<!-- Лучше для независимых скриптов (счетчики, аналитика, реклама) -->
<script async src="analytics.js"></script>
<script async src="ad.js"></script>
<!-- Какой загрузится быстрее — тот и выполнится -->

<!-- ========== defer ========== -->
<!-- НЕ блокирует парсинг HTML, соблюдает порядок, выполняется ПОСЛЕ парсинга HTML -->
<!-- Лучше для скриптов, которым нужен DOM -->
<script defer src="jquery.js"></script>
<script defer src="app.js"></script>
<!-- Всегда сначала jquery.js, потом app.js -->

<!-- ========== Визуальное сравнение ========== -->
<!--
Без атрибутов:
[Загрузка HTML] █████░░░[Скрипт]░░░█████
               ↑ блокировка ↑

async:
[Загрузка HTML] ██████████████
[Скрипт]        ████  (когда загрузился, прерывает парсинг)

defer:
[Загрузка HTML] ██████████████
[Скрипт]                    ████ (после полной загрузки HTML)
-->
```
