# CSS

## 1. Почему для анимации лучше использовать transform?

```js
// Проблема: при анимации свойств left/top/width/height браузеру нужно пересчитывать
// геометрию всей страницы (Reflow), что очень затратно для производительности.

// ========== ПЛОХО (вызывает Reflow + Repaint) ==========
element.style.left = "100px";     // изменяет геометрию
element.style.width = "200px";    // изменяет размер
element.style.top = "50px";       // изменяет позицию
// Браузер: пересчитывает позиции ВСЕХ элементов → Reflow
// Затем перерисовывает → Repaint

// ========== ХОРОШО (только Composite) ==========
element.style.transform = "translateX(100px)";  // не меняет геометрию
element.style.transform = "scale(2)";           // не меняет геометрию
element.style.opacity = "0.5";                  // не меняет геометрию
// Браузер: отправляет элемент на отдельный слой → Composite
// GPU ускоряет анимацию, нет Reflow, нет Repaint

// ========== Что вызывают разные свойства ==========
// Reflow (дорого)      : width, height, left, top, margin, padding, display
// Repaint (средне)     : color, background, border, visibility
// Composite (дешево)   : transform, opacity

// Пример анимации с transform:
@keyframes slide {
    from { transform: translateX(0); }      // ← хорошо
    to   { transform: translateX(100px); }  // ← хорошо
}

// Плохая анимация (реальный пример):
@keyframes badSlide {
    from { left: 0; }   // ← вызывает Reflow на каждом кадре
    to   { left: 100px; }
}
```

## 3. Вес селекторов в CSS (специфичность)

```js
// Чем больше вес селектора, тем выше его приоритет.

// ========== Таблица весов ==========
// !important         → ∞ (самый высокий, старайтесь не использовать)
// Inline стили       → 1000 (style="...")
// ID (#id)           → 100
// Класс (.class)     → 10
// Тег (div, p)       → 1
// Универсальный (*)  → 0
// Псевдоклассы       → 10 (как класс)
// Псевдоэлементы     → 1 (как тег)

// ========== Примеры расчета ==========
// #header .nav li a → 100 + 10 + 1 + 1 = 112
// .menu-item.active → 10 + 10 = 20
// div p → 1 + 1 = 2
// * → 0

// ========== Если веса равны — побеждает последний ==========
<style>
    .title { color: red; }     // вес 10
    .title { color: blue; }    // вес 10, но последний → blue
</style>

// ========== !important перебивает всё (но лучше не использовать) ==========
.text {
    color: red !important;  // перебьет даже inline-стили
}
```

## 5. Относительные единицы в CSS

```css
/* ========== em — относительно родительского шрифта ========== */
.parent {
  font-size: 16px;
}
.child {
  font-size: 1.5em; /* 16px * 1.5 = 24px */
  padding: 2em; /* 24px * 2 = 48px (относительно своего font-size) */
}

/* ========== rem — относительно корневого шрифта (html) ========== */
html {
  font-size: 16px; /* база */
}
.text {
  font-size: 1.5rem; /* 16px * 1.5 = 24px всегда */
  margin: 2rem; /* 32px всегда */
}

/* ========== % — относительно родительского элемента ========== */
.parent {
  width: 500px;
}
.child {
  width: 50%; /* 250px */
  margin-top: 10%; /* 50px (относительно ширины родителя, странно но факт) */
}

/* ========== vw / vh — относительно окна браузера ========== */
.fullscreen {
  width: 100vw; /* 100% ширины окна */
  height: 100vh; /* 100% высоты окна */
}
.half-width {
  width: 50vw; /* 50% ширины окна */
}

/* ========== vmin / vmax — относительно меньшей/большей стороны ========== */
.square {
  font-size: 5vmin; /* 5% от меньшей стороны (width или height) */
}
```
