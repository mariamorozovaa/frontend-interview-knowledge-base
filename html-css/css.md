# CSS

## 1. Почему для анимации лучше использовать transform?

Проблема: при анимации свойств left/top/width/height браузеру нужно пересчитывать геометрию всей страницы (Reflow), что очень затратно для производительности.

```js
//  ПЛОХО (вызывает Reflow + Repaint)
element.style.left = "100px";     // изменяет геометрию
element.style.width = "200px";    // изменяет размер
element.style.top = "50px";       // изменяет позицию
// Браузер: пересчитывает позиции ВСЕХ элементов → Reflow
// Затем перерисовывает → Repaint

//  ХОРОШО (только Composite)
element.style.transform = "translateX(100px)";  // не меняет геометрию
element.style.transform = "scale(2)";           // не меняет геометрию
element.style.opacity = "0.5";                  // не меняет геометрию
// Браузер: отправляет элемент на отдельный слой → Composite
// GPU ускоряет анимацию, нет Reflow, нет Repaint

//  Что вызывают разные свойства
// Reflow (дорого)      : width, height, left, top, margin, padding, display
// Repaint (средне)     : color, background, border, visibility
// Composite (дешево)   : transform, opacity

// Пример анимации с transform:
@keyframes slide {
    from { transform: translateX(0); }      // ← хорошо
    to   { transform: translateX(100px); }  // ← хорошо
}

// Плохая анимация (реальный пример):
@keyframes badSlide {
    from { left: 0; }   // ← вызывает Reflow на каждом кадре
    to   { left: 100px; }
}
```

## 2. Расскажи про позиционирование. Про свойство position и относительно кого происходит позиционирование

Свойство `position` определяет, как элемент располагается на странице и
относительно чего вычисляются его координаты.

```css
position: static | relative | absolute | fixed | sticky;
```

После задания `position` можно использовать:

```css
top;
right;
bottom;
left;
inset;
```

---

## 1. `position: static`

Значение по умолчанию.

- Элемент участвует в нормальном потоке документа.
- `top`, `left`, `right`, `bottom` не работают.
- Позиция определяется соседними элементами.

**Относительно кого?**

Ни относительно кого --- элемент находится в обычном потоке документа.

---

## 2. `position: relative`

Элемент остается в потоке, но его можно сместить.

```css
.block {
  position: relative;
  top: 10px;
  left: 20px;
}
```

- Место под элемент сохраняется.
- Смещение происходит относительно собственной исходной позиции.
- Создает контекст позиционирования для потомков с
  `position: absolute`.

---

## 3. `position: absolute`

Элемент удаляется из нормального потока.

```css
.child {
  position: absolute;
}
```

Особенности:

- Не занимает место в потоке.
- Позиционируется относительно ближайшего предка с `position`,
  отличным от `static`.

Если такого предка нет, используется **Initial Containing Block**
(обычно viewport).

Типичный паттерн:

```css
.parent {
  position: relative;
}

.child {
  position: absolute;
  top: 0;
  right: 0;
}
```

---

## 4. `position: fixed`

Элемент фиксируется относительно окна браузера (viewport).

```css
.button {
  position: fixed;
  right: 20px;
  bottom: 20px;
}
```

- Удаляется из потока.
- Не прокручивается вместе со страницей.

---

## 5. `position: sticky`

Сочетает `relative` и `fixed`.

```css
.header {
  position: sticky;
  top: 0;
}
```

- До достижения `top` ведет себя как `relative`.
- Затем «прилипает».
- Работает относительно ближайшего прокручиваемого контейнера.

---

# Как работают `top`, `right`, `bottom`, `left`

Работают только если `position` не равен `static`.

---

# Почему часто используют `relative + absolute`

```css
.card {
  position: relative;
}

.close {
  position: absolute;
  top: 10px;
  right: 10px;
}
```

Так абсолютно позиционированный элемент привязывается к конкретному
контейнеру, а не к странице.

---

# Сравнение

Position В потоке Относительно чего

---

`static` ✅ Обычный поток
`relative` ✅ Собственной исходной позиции
`absolute` ❌ Ближайшего предка с `position != static`
`fixed` ❌ Viewport
`sticky` ✅ (до прилипания) Ближайшего scroll-контейнера

---

# Частые вопросы

### Почему `absolute` не видит родителя?

Потому что у родителя `position: static`. Добавьте `position: relative`.

### Почему `top` не работает?

Потому что у элемента `position: static`.

### Что происходит с элементом при `absolute`?

Он удаляется из нормального потока, поэтому остальные элементы
располагаются так, как будто его нет.

### Чем отличается `fixed` от `sticky`?

- `fixed` всегда привязан к viewport.
- `sticky` становится фиксированным только при прокрутке.

---

# Что сказать на собеседовании

> `position` определяет способ позиционирования элемента. `relative`
> смещает элемент относительно собственной позиции и создает контекст
> для `absolute`. `absolute` позиционируется относительно ближайшего
> предка с `position != static`. `fixed` привязан к viewport, а `sticky`
> сочетает поведение `relative` и `fixed`, фиксируя элемент при
> прокрутке.

## 3. Вес селекторов в CSS (специфичность)

Чем больше вес селектора, тем выше его приоритет.

```js
// Таблица весов
// !important → ∞ (самый высокий, старайтесь не использовать)
// Inline стили → 1000 (style="...")
// ID (#id) → 100
// Класс (.class) → 10
// Тег (div, p) → 1
// Универсальный (\*) → 0
// Псевдоклассы → 10 (как класс)
// Псевдоэлементы → 1 (как тег)

// Примеры расчета
// #header .nav li a → 100 + 10 + 1 + 1 = 112
// .menu-item.active → 10 + 10 = 20
// div p → 1 + 1 = 2
// \* → 0

// Если веса равны — побеждает последний

<style>
    .title { color: red; }     // вес 10
    .title { color: blue; }    // вес 10, но последний → blue
</style>

//  !important перебивает всё (но лучше не использовать)
.text {
    color: red !important;  // перебьет даже inline-стили
}
```

## 5. Относительные единицы в CSS

```css
/*  em — относительно родительского шрифта  */
.parent {
  font-size: 16px;
}
.child {
  font-size: 1.5em; /* 16px * 1.5 = 24px */
  padding: 2em; /* 24px * 2 = 48px (относительно своего font-size) */
}

/*  rem — относительно корневого шрифта (html)  */
html {
  font-size: 16px; /* база */
}
.text {
  font-size: 1.5rem; /* 16px * 1.5 = 24px всегда */
  margin: 2rem; /* 32px всегда */
}

/*  % — относительно родительского элемента  */
.parent {
  width: 500px;
}
.child {
  width: 50%; /* 250px */
  margin-top: 10%; /* 50px (относительно ширины родителя, странно но факт) */
}

/*  vw / vh — относительно окна браузера  */
.fullscreen {
  width: 100vw; /* 100% ширины окна */
  height: 100vh; /* 100% высоты окна */
}
.half-width {
  width: 50vw; /* 50% ширины окна */
}

/*  vmin / vmax — относительно меньшей/большей стороны  */
.square {
  font-size: 5vmin; /* 5% от меньшей стороны (width или height) */
}
```
