# Scope, Hoisting, Global Object, Strict Mode

## Теория (вопросы и ответы)

---

## Какие области видимости есть?

В JavaScript существует несколько областей видимости (Scope).

### 1. Глобальная область видимости (Global Scope)

Переменные доступны из любого места программы.

```js
const name = "Maria";

function sayHi() {
  console.log(name);
}
```

---

### 2. Функциональная область видимости (Function Scope)

Переменные, объявленные через `var`, существуют только внутри функции.

```js
function test() {
  var x = 10;
}

console.log(x); // ReferenceError
```

---

### 3. Блочная область видимости (Block Scope)

Переменные `let` и `const` существуют только внутри блока `{}`.

```js
if (true) {
  const name = "Maria";
}

console.log(name); // ReferenceError
```

Блоками являются:

- if
- for
- while
- switch
- просто `{}`

---

### 4. Модульная область видимости (Module Scope)

Каждый ES-модуль (`import/export`) имеет собственную область видимости.

Переменные одного модуля недоступны в другом без явного экспорта.

---

## Что такое Scope?

Scope — это область видимости, которая определяет, где переменная доступна.

Например:

```js
const a = 10;

function foo() {
  const b = 20;

  console.log(a);
  console.log(b);
}
```

Внутри функции доступны:

- локальные переменные;
- переменные внешних областей видимости.

А вот внешняя область не видит локальные переменные.

```js
console.log(b); // ReferenceError
```

---

## Как движок понимает, что переменную нужно искать во внешнем Scope?

Во время создания функции JavaScript сохраняет ссылку на окружающую область видимости.

Она хранится во внутреннем скрытом свойстве:

```text
[[Environment]]
```

Именно благодаря ему работают замыкания.

Например:

```js
const a = 10;

function foo() {
  console.log(a);
}
```

При создании функции:

```text
foo
 │
 ▼
[[Environment]]
 │
 ▼
Global Scope
```

Когда выполняется

```js
foo();
```

движок ищет переменную так:

```text
Локальная область

↓

Внешняя область ([[Environment]])

↓

Еще выше

↓

Global Scope
```

Этот процесс называется **Scope Chain** (цепочка областей видимости).

Если переменная нигде не найдена:

```text
ReferenceError
```

---

## Куда попадают функции и переменные, объявленные в глобальной области видимости?

Все зависит от способа объявления.

### var

Попадает в глобальный объект.

```js
var a = 10;

console.log(window.a); // 10
```

---

### let и const

Не становятся свойствами глобального объекта.

```js
const b = 20;

console.log(window.b); // undefined
```

Хотя переменные остаются глобальными, они находятся в отдельной области хранения.

---

## Что такое глобальный объект?

Глобальный объект содержит встроенные функции и глобальные свойства среды выполнения.

В браузере это

```js
window;
```

В Node.js

```js
global;
```

Современный универсальный вариант

```js
globalThis;
```

Например:

```js
window.setTimeout();

window.console;

window.document;
```

или

```js
globalThis.setTimeout();
```

---

## Что такое window?

`window` — глобальный объект браузера.

Он содержит:

- document
- location
- history
- navigator
- localStorage
- sessionStorage
- console
- setTimeout
- setInterval

Практически всё API браузера находится внутри `window`.

---

## Что такое Hoisting?

Hoisting — это механизм, при котором объявления переменных и функций обрабатываются движком до выполнения кода.

Важно понимать:

JavaScript **не переносит код вверх**, а заранее создает объявления во время фазы создания Execution Context.

---

### Hoisting для var

```js
console.log(a);

var a = 10;
```

Фактически выглядит как:

```js
var a;

console.log(a);

a = 10;
```

Ответ:

```js
undefined;
```

---

### Hoisting для let и const

```js
console.log(a);

let a = 10;
```

Ответ:

```text
ReferenceError
```

Переменная уже существует, но находится в **Temporal Dead Zone (TDZ)**.

---

### Hoisting функций

Function Declaration можно вызвать до объявления.

```js
sayHi();

function sayHi() {
  console.log("Hello");
}
```

Работает корректно.

---

Function Expression

```js
sayHi();

const sayHi = function () {};
```

Получим

```text
ReferenceError
```

---

## Что такое необъявленные переменные?

Необъявленная переменная — это переменная, которая используется без `let`, `const` или `var`.

Например:

```js
name = "Maria";
```

Без строгого режима:

переменная автоматически станет глобальной.

Со строгим режимом:

```text
ReferenceError
```

---

### Необъявленная и неинициализированная переменная

Необъявленная:

```js
console.log(x);
```

```text
ReferenceError
```

Вообще не существует.

---

Объявленная, но неинициализированная:

```js
let x;

console.log(x);
```

Ответ:

```js
undefined;
```

---

## Что такое "use strict"?

```js
"use strict";
```

Включает строгий режим JavaScript.

Он делает язык более безопасным.

Например:

Без строгого режима:

```js
name = "Maria";
```

Работает.

Со строгим режимом:

```text
ReferenceError
```

---

### Что запрещает strict mode?

#### Необъявленные переменные

```js
x = 10;
```

Ошибка.

---

#### Дублирование параметров

```js
function foo(a, a) {}
```

Ошибка.

---

#### Запись в read-only свойства

```js
Object.freeze(obj);
```

Попытка изменить объект вызовет ошибку.

---

#### this

В обычной функции

Без strict:

```js
this === window;
```

Со strict:

```js
this === undefined;
```

## Что сказать на собеседовании

> Scope определяет область видимости переменных. JavaScript использует лексическую область видимости: при создании функции сохраняется ссылка на внешнее окружение (`[[Environment]]`), благодаря чему работает Scope Chain и замыкания. Объявления переменных и функций обрабатываются на этапе создания Execution Context (Hoisting), при этом `var` и Function Declaration ведут себя по-разному по сравнению с `let`, `const` и Function Expression. В браузере глобальным объектом является `window`, а универсальным способом обращения к глобальному объекту считается `globalThis`. Строгий режим (`"use strict"`) делает код безопаснее, запрещая ряд потенциально опасных конструкций.
