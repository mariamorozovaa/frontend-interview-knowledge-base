## 2. Как работают for...in, for...of, await for?

---

### for...in

Итерируется по **ключам объекта** (и наследуемым тоже).

```js id="m1n2o3"
const obj = { a: 1, b: 2 };

for (let key in obj) {
  console.log(key);
}
```

⚠️ Не рекомендуется для массивов.

---

### for...of

Итерируется по **значениям итерируемых объектов**.

```js id="p4q5r6"
const arr = [1, 2, 3];

for (let value of arr) {
  console.log(value);
}
```

Работает с:

- массивами

- строками

- Map

- Set

---

### await for (for await...of)

Используется для **асинхронных итераторов**.

```js id="s7t8u9"
for await (const chunk of stream) {
  console.log(chunk);
}
```

Применяется в:

- streaming API

- async generators

## 3. Из чего состоит цикл for?

---

### Синтаксис:

```js id="v1w2x3"
for (инициализация; условие; шаг) {
  // тело цикла
}
```

---

### Части:

| часть | обязательность | описание |

|------|----------------|----------|

| инициализация | ❌ | выполняется 1 раз |

| условие | ✔ | проверяется перед каждой итерацией |

| шаг | ❌ | выполняется после каждой итерации |

---

### Пример:

```js id="y4z5a6"
for (let i = 0; i < 5; i++) {
  console.log(i);
}
```

---

### Можно пропускать части:

```js id="b7c8d9"
let i = 0;

for (; i < 5; ) {
  i++;
}
```

---

## 4. Как работает while?

---

`while` выполняет цикл, пока условие true.

```js id="e1f2g3"
let i = 0;

while (i < 5) {
  console.log(i);

  i++;
}
```

---

### Особенность:

- сначала проверяет условие

- потом выполняет тело

---

### Бесконечный цикл:

```js id="h4i5j6"
while (true) {
  break;
}
```

---

## 5. Почему лучше не изменять массив во время for?

---

Потому что это ломает логику итерации.

---

### Пример проблемы:

```js id="k7l8m9"
const arr = [1, 2, 3, 4];

for (let i = 0; i < arr.length; i++) {
  arr.pop();
}
```

---

### Что происходит:

- length меняется

- индексы сдвигаются

- элементы пропускаются

---

### Итог:

- пропуск элементов

- баги в логике

- непредсказуемое поведение

---

## 6. reduce + {} как начальное значение

---

### Пример:

```js id="n1o2p3"
const result = [1, 2, 3].reduce((acc, val) => {
  acc[val] = val;

  return acc;
}, {});
```

---

### Что будет?

- acc — объект

- можно использовать как словарь

---

### Итог:

```js

{ 1: 1, 2: 2, 3: 3 }

```

---

### Важно:

Если НЕ вернуть acc → будет ошибка логики или undefined.

---

## 7. Что если в reduce убрать return?

---

```js id="q4r5s6"
[1, 2, 3].reduce((acc, val) => {
  acc += val;

  // нет return ❌
}, 0);
```

---

### Что произойдёт?

- acc станет undefined на следующей итерации

- reduce сломается

---

### Ошибка:

```text

TypeError: Cannot read properties of undefined

```

## 8. Что такое SET и MAP?

---

### Set

Коллекция **уникальных значений**.

```js id="t1u2v3"
const set = new Set([1, 2, 2, 3]);

console.log(set); // {1,2,3}
```

---

### Map

Коллекция **ключ-значение (любые типы ключей)**.

```js id="w4x5y6"
const map = new Map();

map.set("a", 1);

map.set({}, 2);
```

---

### Отличия от Object:

- ключи могут быть объектами

- сохраняется порядок

- легче итерировать

---

## 9. Map vs WeakMap

---

| Map | WeakMap |

|-----|--------|--------|

| любые ключи | только объекты |

| iterable | не iterable |

| влияет на GC | не мешает GC |

| хранит ссылки | слабые ссылки |

---

### WeakMap пример:

```js id="z7a8b9"
let obj = {};

const wm = new WeakMap();

wm.set(obj, "data");

obj = null; // объект удалится GC
```

---

## 10. Почему Map легче итерируется, чем Object?

---

### Map:

- гарантированный порядок

- встроенный iterator

- нет лишних ключей

```js id="c1d2e3"
for (let [k, v] of map) {
  console.log(k, v);
}
```

---

### Object:

- может иметь prototype keys

- нет гарантированного порядка (исторически)

- нужно Object.keys()

---

### Вывод:

Map — нативно предназначен для итераций

Object — универсальная структура, не оптимизированная под это

---

## 11. Как понять, что объект итерируемый?

---

Объект итерируемый, если у него есть:

```js id="f4g5h6"
obj[Symbol.iterator];
```

---

### Проверка:

```js id="i7j8k9"
console.log(typeof obj[Symbol.iterator] === "function");
```

---

### Примеры:

- Array

- String

- Map

- Set

---

## 12. Как сделать любой объект перебираемым?

---

Нужно добавить `[Symbol.iterator]`.

---

### Пример:

```js id="l1m2n3"
const obj = {
  a: 1,

  b: 2,

  [Symbol.iterator]() {
    const values = Object.values(this);

    let index = 0;

    return {
      next: () => ({
        value: values[index++],

        done: index > values.length,
      }),
    };
  },
};
```

---

### Использование:

```js id="o4p5q6"
for (let value of obj) {
  console.log(value);
}
```

---

## 13. Что такое spread / rest и как работает?

---

### Spread оператор (...)

Разворачивает структуру.

```js id="r7s8t9"
const arr = [1, 2];

const newArr = [...arr, 3];
```

---

### Rest оператор

Собирает значения в массив.

```js id="u1v2w3"
function sum(...args) {
  return args.reduce((a, b) => a + b);
}
```

---

### В объектах:

```js id="x4y5z6"
const obj = { a: 1, b: 2 };

const copy = { ...obj };
```

---

### Как примерно работает:

- spread → перебирает через iterator

- rest → собирает остаток аргументов

---

## Что сказать на собеседовании

Массивы в JS имеют набор методов для трансформации, фильтрации и агрегации данных.
Циклы for, for...of и for...in работают с разными типами данных и уровнями абстракции.
for...of работает через итераторы, а for...in — по ключам объекта.
Map и Set — современные структуры данных с поддержкой итерации и предсказуемым поведением.
Map удобнее для обхода, чем Object, так как он специально создан для хранения пар ключ-значение.
Итерируемость определяется наличием Symbol.iterator.
Любой объект можно сделать итерируемым, добавив этот метод.
Spread и rest работают через механизм итераторов и позволяют удобно разбирать и собирать данные.
